# Contributing Guide

Thank you for your interest in contributing to Instagram Red Flags.

## Ways to Contribute

- Report bugs
- Propose features
- Improve documentation
- Submit code fixes and enhancements

## Development Setup

1. Fork the repository.
2. Clone your fork.
3. Create a virtual environment.
4. Install dependencies from `requirements.txt`.
5. Run the app with `python main.py`.

## Branch and Commit Guidelines

- Create feature branches from `main`.
- Keep pull requests focused and small where possible.
- Use clear commit messages describing intent.

## Pull Request Requirements

- Explain what changed and why.
- Include test or validation notes.
- Update docs when behavior changes.
- Ensure no personal data files are included.

## Coding Standards

- Keep code readable and modular.
- Use clear naming and concise comments.
- Preserve existing architecture patterns.

## Reporting Bugs

When filing a bug, include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Environment (OS, Python version)
- Screenshots/logs when helpful

## Security Issues

Do not open public issues for sensitive vulnerabilities. See [SECURITY.md](SECURITY.md) for reporting instructions.
